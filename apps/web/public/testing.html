How to use this page

Two parts: a manual QA checklist (below) and a Demo Test Data section at the bottom with copy buttons. Work top-to-bottom; each checklist item states the expected result in green.

Test on a non-production workspace where possible. For payments, always use PayPal Sandbox accounts — never a real card. Resize the browser (or use device emulation) to confirm mobile layouts.

1 · Authentication

  • Register a new account with a valid email + strong password.Expected: account created, redirected into the app; a verification path is available.
  • Log out, then log back in with the same credentials.Expected: login succeeds and lands on the dashboard.
  • Attempt login with a wrong password.Expected: a clear error, no token issued.
  • Use "Forgot password" → request reset.Expected: request accepted with a neutral confirmation (no account enumeration).
  • Leave the app idle until the access token expires, then act.Expected: token refreshes silently; you stay signed in.
  • Visit an app URL (e.g. /dashboard) while logged out.Expected: redirected to login.

2 · Workspaces, roles & isolation

  • Create a second workspace and switch between them.Expected: the switcher changes context; data does not leak between workspaces.
  • Invite a teammate and assign a role (Admin / Marketer / Analyst / Viewer).Expected: invite sent; role saved.
  • As a non-Owner, try an Owner-only action (e.g. start checkout).Expected: 403 / blocked with a clear message.
  • As a Viewer, confirm write actions are hidden or disabled.Expected: read-only experience.
  • Try to open another workspace's record by ID in the URL.Expected: not found / forbidden — never another tenant's data.

3 · Onboarding & Growth DNA

  • Complete the onboarding wizard using the demo business profile.Expected: all steps save; you reach the finish state.
  • Generate the Growth DNA profile.Expected: a saved profile with summary, readiness scores, and a 30-day plan.
  • Regenerate / create a second profile, then reopen an earlier one.Expected: history library lists versions; reopening restores its answers.
  • Rename and delete a saved profile.Expected: label updates; delete removes it from the library.

4 · Agents & runs

  • From the Agents dashboard, run a skill agent.Expected: a run is created with status; output is saved and traceable to the workspace.
  • Open the run detail page.Expected: inputs, outputs, model, and token/cost info are shown.
  • Exceed the plan's monthly AI-credit pool with repeated runs.Expected: a 402 with a clear upgrade message — nothing runs silently over the cap.

5 · Recommendations & approvals

  • Open a recommendation card.Expected: shows what/why, expected impact, risk level, and related platform/campaign.
  • Approve, reject, and edit-before-applying a recommendation.Expected: status changes; an audit-log entry records the action.
  • Confirm a sensitive action (budget/launch/tracking) requires approval by default.Expected: it is gated unless Autopilot is explicitly enabled with limits.

6 · Billing & PayPal payments

Recurring plans are billed by PayPal (Paddle removed). Use PayPal Sandbox — see the demo data below.

  • Open Settings → Billing. Confirm plan cards, usage meters, and (if unconfigured) the "not configured yet" notice reference PayPal.Expected: no Paddle references anywhere.
  • With PayPal configured (sandbox), click Upgrade on a plan.Expected: full-page redirect to PayPal approval (not an overlay).
  • Approve in the PayPal sandbox and return.Expected: land on /billing?checkout=success with a success banner.
  • Wait for the BILLING.SUBSCRIPTION.ACTIVATED webhook (or fire it from PayPal's simulator).Expected: plan flips to active; billing status reflects it.
  • Re-deliver the same webhook event id.Expected: idempotent no-op (no duplicate state change).
  • Click "Manage plan".Expected: opens PayPal's autopay page to view/cancel.
  • Cancel, then fire BILLING.SUBSCRIPTION.CANCELLED.Expected: reverts to free tier limits; data preserved.
  • Cancel the checkout on PayPal instead of approving.Expected: land on ?checkout=cancelled; no charge, no plan change.

7 · Integrations / OAuth

  • Open Settings → Integrations.Expected: real connection status per provider; disconnected ones show a Connect button.
  • Start an OAuth connect flow for a provider.Expected: redirects to the provider's consent screen; tokens are never shown in the UI.
  • Disconnect a connected provider.Expected: status returns to disconnected; a sync/audit record exists.

8 · Modules — Campaigns, Traffic, SEO, Website, Reports

  • Campaigns: view synced campaign records (or the empty state when nothing is connected).Expected: real records or a clean empty state — never fabricated metrics.
  • Traffic Genie: build a UTM link with the demo values.Expected: a correctly-tagged URL is produced.
  • SEO & GEO: run an audit using the demo domain/keywords.Expected: opportunities/recommendations are saved.
  • Website: audit the demo landing page URL.Expected: a health/conversion score plus recommendations.
  • Reports: generate a report.Expected: built from real data/saved outputs; PDF export works.

9 · Help center (Text / Audio / Video)

  • Open Help from the sidebar and pick a topic (e.g. Billing & plans).Expected: the topic list is grouped by category; the article renders on the Text tab.
  • With ElevenLabs configured, open the Audio tab.Expected: "Preparing narration…", then an audio player; re-opening plays the cached file instantly.
  • Without an ElevenLabs key, open the Audio tab.Expected: a clean "coming soon" state — no error.
  • Open the Video tab.Expected: a "Coming soon" panel (labeled), no broken player.

10 · Empty states

  • View Campaigns / Reports / Email before connecting anything.Expected: helpful empty states with clear CTAs — no fake charts or numbers.

11 · Mobile responsiveness

  • Resize to a phone width (≈375px) across dashboard, billing, and help.Expected: no horizontal scroll; bottom nav appears; tables/cards collapse gracefully.
  • On the Help page, confirm the two-pane layout stacks on mobile.Expected: topic list above the article; tabs remain usable.

12 · Security

  • Call a workspace API route with no / another workspace's token.Expected: 401 / 403 — auth + isolation enforced.
  • POST the PayPal webhook with a bad/missing signature.Expected: 400, nothing applied.
  • Confirm OAuth/provider tokens are never returned to the browser.Expected: encrypted at rest; only status is exposed.

Billing — PayPal Sandbox (no real credentials)
1. Create sandbox accounts at https://developer.paypal.com → Sandbox → Accounts
   (one BUSINESS = merchant, one PERSONAL = buyer).
2. Create a Product + Billing Plans (monthly + annual) and copy the Plan IDs
   into PAYPAL_PLAN_ID_STARTER / _PRO / _AGENCY (and *_ANNUAL).
3. Set PAYPAL_ENVIRONMENT=sandbox and the sandbox CLIENT_ID / CLIENT_SECRET.
4. Register the webhook → https://<api-host>/api/v1/billing/paypal/webhook and
   copy its Webhook ID into PAYPAL_WEBHOOK_ID.
5. At checkout, log in with the sandbox BUYER account to approve.
6. Use Developer Dashboard → Webhooks → "Simulate" to fire
   BILLING.SUBSCRIPTION.ACTIVATED / CANCELLED and verify the plan updates.
Never use a real PayPal account or card for testing.
